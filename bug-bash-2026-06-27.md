# BUG BASH 2026-06-27 — open-bug triage into 3 worker lanes

A triage of every open **bug** in [`claude-todo.md`](claude-todo.md) (a *defect*: wrong-code /
crash / invalid-IR / wrong-output / cross-mode mis-marshal / a checker under-enforcement or
over-rejection), split into three **fairly-orthogonal lanes** so three workers can run in
parallel without colliding. Non-bug items (design/discussion, planning, perf, opportunistic
refactors, pure test-coverage, doc/spec authoring, BUILDER-lag cleanups, hygiene) are **not**
listed here.

The split axis is **the file-tree each lane owns** — that is what actually prevents two workers
stepping on each other:

| Lane | Owns | Theme |
|---|---|---|
| **1 — front-end semantics** | `pkg/binate/{checker,types,parser}` | *wrongly accepts / wrongly rejects / wrong diagnostic* |
| **2 — IR-gen & native codegen** | `pkg/binate/{ir,codegen,native/*}` | *emits wrong or invalid code* |
| **3 — VM & cross-mode runtime** | `pkg/binate/vm`, `pkg/std`, ABI marshaling | *interpreter / cross-mode marshaling wrong results; 32-bit-host latents* |

Legend: 🔴 critical · 🟠 major · 🟡 minor · ⚠ needs a **semantics/scope decision** before a
worker can just code it (get a ruling first) · 🔶 large / user-deferred — not a quick bash, flag
before starting · 🤝 shared-file coupling with another lane — coordinate.

> **Snapshot caveat.** Entries are referenced by their `(subsystem) — symptom` heading (grep the
> phrase in `claude-todo.md`), **not** by line number — the file churns under concurrent workers.
> Accurate as of `explorations` commit `aa224cab`. Two formerly-listed bugs (named-managed-slice
> struct field on native + the VM uninitialized-read facet) were **resolved** (`214db9bf` /
> `c7d9ffca`) and folded into the Lane-2 *un-peel sweep* below.

## LANE 1 — front-end: type-checker, type-system & parser

`pkg/binate/checker`, `pkg/binate/types`, `pkg/binate/parser`. The "wrongly accepts / wrongly
rejects / wrong diagnostic" family. (Largest lane, but heavy on MINOR + decision-gated items —
the genuinely codeable-now count is smaller.)

- 🔴 **(checker / silent integer-wrap)** `:=` short-var does NOT fit-check an over-wide literal →
  silently wraps `0xFFFFFFFFFFFFFFFF` to `int -1` (explicit `var x int = …` correctly rejects).
- 🔴 ⚠ **(type-checker / `conv.no-implicit-numeric`)** distinct same-width integer types implicitly
  inter-convert (`int ↔ int64`); `Type.Identical` INT arm ignores the name. User-confirmed bug;
  **broad blast radius** — needs an int↔int64-cast sweep tree-wide; scope it before starting.
- 🟠 **(entry / under-enforcement)** `main` signature not enforced (`func main(x int)` /
  `func main() int` compile + link + run).
- 🟠 **(type-checker / assignability — cross-pkg named scalar)** cross-package distinct named
  *scalar* types wrongly inter-assign (`red.A` → `blue.B`, no cast); same-package correctly rejects.
- 🟠 🤝 **(type-checker / assignability — sub-interface)** impl of a *sub*-interface not assignable
  to its *super* (`var b *Base = &r` rejected though `R impl Sub : Base`). **Lane 2 coupling**:
  needs a companion IR-gen ancestor-closure record — fix checker + IR together (likely the same
  imported-impl path as the Lane-2 third-pkg-vtable bug).
- 🟠 **(latent — generic interface collision)** generic *interfaces* with the same final path
  segment collide (short-name keying); mirror the landed struct/func fix → key on full package path.
- 🟠 **(named func-value literal)** `var f Fn = func(...){...}` rejected for a named func-value type;
  3 checker sites need `TYP_NAMED` peeling (🤝 +1 IR site — mild Lane 2 coupling).
- 🟠 **(generic constraint unchecked)** generic struct/interface instantiation skips
  constraint-satisfaction (`Box[NoOrder]` compiles though `T lang.Orderable`); generic-*function*
  path already correct.
- 🟠 ⚠ **(X3-highbit — const-fold divergence)** signed sign-bit const-fold checker-vs-IR divergence
  (`1<<63` to signed: checker rejects, IR wraps to INT_MIN). Spec-owned — get a ruling.
- 🟡 **(type-checker / primitive-impl carve-out)** §11.10 not enforced in the impl-collection pass
  (a non-`lang` package may `impl int : <empty iface>`).
- 🟡 **(lexical / char literal)** single-byte char-literal constraint not enforced (`''` → 0x00;
  `'ab'` → first byte). (lexer/checker.)
- 🟡 **(checker — duplicate imports)** duplicate same-short-name imports accepted silently; `pkg.X`
  resolves first-wins.
- 🟡 **(checker — unused imports)** checker has zero unused-import handling (only bnlint, with
  cross-file FP/FN corners); depends on the file-scoped-imports work.
- 🟡 **(checker — bad-key diagnostic)** keyed struct-literal bad key reports generic "undefined"
  not "no field … in …".
- 🟡 **(checker — positional struct literal)** positional struct-literal elements skip the
  assignability check (keyed ones get it).
- 🟡 ⚠ **(checker — opaque field access)** direct field access on an opaque cross-pkg type not
  rejected (may be a test-setup artifact — re-check with a pure-`.bni` setup first).
- 🟡 ⚠ **(checker — value-receiver readonly)** value-receiver "always readonly" not enforced (no
  correctness bug — the copy makes mutation harmless; decide enforce vs downgrade the note).
- 🟡 ⚠ **(checker — untyped-const coercion)** untyped-const coercion diverges from the DECIDED note
  (coerces like a literal); decide enforce-note vs accept-and-update-notes.
- 🟡 ⚠ **(checker — inferred-length array)** inferred-length `[...]T{...}` not implemented (checker
  rejects); wire it or mark deferred.
- 🟡 ⚠ **(checker — single-member iota)** `iota` not resolved in a *single-member* grouped const
  block (impl needs 2+).
- 🟡 ⚠ **(parser — generic-body concrete instantiation)** a concrete-instantiation parameter type
  in a `.bni` body fails to parse (`func Sum(x @Box[int])`); the generic-reader idiom works.
- 🟡 ⚠ **(checker — generic-body type-param arithmetic)** a generic-body expr mixing a type-param
  field/index with arithmetic mis-diagnoses ("requires numeric operands").
- 🟡 **(parser — inferred-length composite)** `[...]T{...}` array composite literal rejected at

## LANE 2 — IR-gen & native code generation

`pkg/binate/ir`, `pkg/binate/codegen`, `pkg/binate/native/{x64,aarch64,arm32,common}`. The "emits
wrong or invalid code" family.

**IR-gen / shared codegen:**

- 🟠 **(IR-gen / un-peeled element type)** — 🟡 IN PROGRESS, OWNER ACTIVE (`aa224cab`). Slice/pointer
  indexed-assign + for-range arms read `collection.Typ.Elem` un-peeled → silent wrong-stride /
  wrong-refcount for named-distinct slices (and native nil-element-GEP SIGSEGV). Slice-set arm done
  (`c7d9ffca`); remaining sites listed in the todo. **Coordinate — don't double-up.**
- 🔴 **(codegen / invalid IR)** `switch` on a sub-64-bit tag with an untyped-literal case emits
  invalid IR (`i64` literal vs `iN` tag → LLVM verifier error); narrow each case constant to the
  tag's type.
- 🔴 **(IR-gen / runtime negative shift)** runtime-count negative shift lowers `lshr` instead of
  `ashr` (opcode picked from the operand TYPE's Signed flag, not the value's sign) — needs a
  value-based decision for untyped const operands.
- 🟠 **(IR-gen / method-expr named scalar)** method *expression* over a named *scalar* type
  miscompiles: direct form → undefined `@bn_Celsius__Plus`; `*func` form → SIGSEGV. Fails on LLVM
  **and** VM (shared front-of-backend, not LLVM-only).
- 🟠 🤝 **(IR-gen / third-pkg impl vtable)** cross-pkg `impl R : I` declared only in a *third*
  package: checker accepts but IR-gen never wires the (R,I) vtable → null-vtable SIGSEGV at
  dispatch. **Lane 1 coupling** — likely the same imported-impl path as the sub-interface bug.
- 🟠 **(IR-gen / grouped-signedness const)** a grouped-signedness const followed by a bare repeat
  loses its stamp to a shared-node re-check → folds signed (latent wrong-code).
- 🟠 **(IR-gen / iota `.bni` consts)** iota-grouped `.bni` consts stay value-less (substitution not
  ported into `bni_scope`) → a negative iota-grouped imported const used as a shift count slips
  unchecked.
- 🟠 ⚠ **(IR-gen / indexed array literal)** indexed array literals silently MISCOMPILE
  (`[5]int{1:10,3:30}` → `{10,30,0,0,0}`, keys ignored). DECIDED behavior exists; fix-or-reject is
  the user's call.
- 🟠 ⚠ **(checker+IR / generic aggregate comparison)** generic instantiation can reach the
  invalid-IR aggregate-comparison class (body checked once with `T` opaque; instantiation skips the
  re-check). Needs instantiation-time re-check OR a `comparable`-style constraint.
- 🟡 **(IR-gen / managed ptr-to-array index)** `(*m)[i]` on `@([3]int)` → "invalid getelementptr
  indices"; the raw `*([N]T)` form works.

**Native backends:**

- 🔴 **(native x64 codegen / inline-capture clobber)** ROOT-CAUSED. Caller's outgoing stack-args
  overlap an inline-capture local (frame allocator doesn't reserve max outgoing-args below locals)
  → silent capture corruption; aarch64 OK. Exposed by conformance 926.
- 🔴 **(arm32 codegen / silent wrong compare)** `int64 == <inline untyped negative const-expr>`
  miscompiled (false where true) — const-expr operand not sign-extended to 64-bit before the
  32-bit-pair compare.
- 🟠 **(native-aa64 codegen / named managed-slice)** a distinct named type over a managed-slice
  (`type Buf @[]int`) miscompiles index/len/slice/assign on native-aa64 only (LLVM/VM/arm32/gen1/
  gen2 correct).
- 🟠 **(native / arm32 Package-accessor ResultSize)** Package-accessor `ResultSize` hardcoded to
  `ptrSize` → target-independence break on ILP32 (per-mode expected overrides; arm32 CI gated on
  qemu).
- 🟡 **(native x64 / iface-call HFA)** native iface-call `emitCallIfaceMethod` lacks HFA
  classification (struct-of-floats arg mis-seen as a GP aggregate).
- 🟡 **(types/ir/native/vm — funcval/iface field order)** funcval `{vtable,data}` / iface
  `{data,vtable}` field orders hardcoded as magic indices (agree by convention only) — harden into
  shared named offsets in `pkg/binate/types`. Cross-cutting hardening, not a live miscompile.
- 🟡 **(native / `_Package` accessor ILP32 hardcode)** native `_Package()` accessor hardcodes the
  16-byte managed-header offset (LP64-only; `16 == 2*IntSize` today). Not a live bug — fixes when a
  32-bit *native* target lands.
- 🟡 **(native-common — byte order)** little-endian only; `TargetInfo` carries no endianness field
  (future big-endian/cross-endian targets).
- 🟠 🔶 **(PROJECT — relro section infra)** Mach-O `__DATA_CONST,__const` / ELF `.data.rel.ro` for
  relocatable read-only blobs (now in writable `data`). Defense-in-depth, not blocking; **large
  infra**, flag before starting.

## LANE 3 — VM / interpreter & cross-mode runtime

`pkg/binate/vm`, `pkg/std`, ABI marshaling. The "interpreter / cross-mode marshaling wrong results;
32-bit-host latents" family.

- 🔴 **(VM / float64 marshal)** `stdlib/math` float64 classification mis-marshals across the
  VM↔native boundary on an x86-64 *host* VM only (aarch64 host + all compiled backends correct).
- 🔴 **(VM / stat-isdir double-interp)** `os.Stat(file).IsDir()` wrongly reports a regular file as a
  directory under VM-on-VM (`builder-comp-int-int`), breaking `cmd/bni` `expandDirArgs`; regression
  from 87a3544a.
- 🔴 🤝 **(VM + native-aa64 / bit_cast sub-word)** `bit_cast` to a sub-word type not narrowed in the
  VM (and **native-aa64**) — leaks high bits (`-1` instead of `255`/`4294967295`); LLVM/arm32
  narrow correctly. **Lane 2 coupling** (the aa64 facet is a native-codegen fix; the VM facet is
  here).
- 🔴 🔶 **(VM / bytecode `__Package` descriptor)** VM bytecode has no per-package `__Package()` body
  (only the 4 builtins via hardcoded externs) → "extern not found: <pkg>.__Package" for user/stdlib
  packages. **Large VM-backend work, user-deferred** — flag before starting.
- 🟠 **(VM / `os.Stat().ModTime`)** `ModTime()` returns sec ≤ 0 under the bytecode VM
  (`builder-comp-int` only) — likely cross-mode marshaling of the 16-byte `time.Point {i64,i32}`
  struct return.
- 🟠 ⚠ **(c-call / latent ABI)** `__c_call` returning a C `int` (32-bit) read as a binate `int`
  (64-bit) is UB on x86-64 (`-1` → 4294967295). No known live instance; decide checker-reject vs
  coding-guide note.
- 🟠 **(VM / register stride 32-bit)** VM register-alloc stride latent on a 32-bit host (8-byte slot
  budget vs 4-byte `int` → registers alias). Latent; blocks 32-bit-hosted VM.
- 🟠 **(VM / int↔float 32-bit)** VM `int64↔float` conversion not pair-aware on 32-bit
  (`BC_SITOF`/`BC_FTOSI` drop/leave a half). Latent; blocks arm32 VM.
- 🟡 **(VM / cross-mode iface dispatch — untested shapes)** VALUE-receiver, multi-aggregate args,
  float arg, n>6 overflow untested — latent defect in VALUE-receiver result handling + coverage gap.
- 🟡 **(VM / cross-mode iface — ILP32 resultSize)** `dispatchCompiledIfaceMethod` resultSize
  threshold is LP64-only; an ILP32 host picks the wrong shim (latent, host-only).
- 🟡 **(VM / cross-mode iface — 32-bit argslots)** 64-bit-scalar args pack as 2 slots on a 32-bit
  host, out of sync with dispatch's positional arg reads (latent, host-only).
- 🟡 **(VM / iface upcast offset>0)** offset>0 upcast dispatch unreachable (`lookupShimVtable` is
  exact-match; needs a range lookup). Unreachable today (no runtime iface extends another).
- 🟡 **(stdlib / float NaN total order)** float32/float64 `Compare`/`Hash` don't implement IEEE
  total order at NaN (violates the Orderable/Hashable promise). (`pkg/std` / `pkg/builtins/lang`.)
- 🔴 (coverage) **(abi-matrix managed component)** abi-matrix multi-return-through-dispatch cells
  lack a managed-component type (`@[]T`/`@T`), missing the Axiom-3 copy-RefInc end-to-end
  validation. Coverage gap, not a live miscompile — regenerate the matrix + add refcount-balance.

## Cross-lane couplings (read before parallelizing)

- **Lane 1 ⇄ Lane 2** share `pkg/binate/ir` for the interface-vtable work: the *sub-interface
  assignability* (Lane 1) and *third-package impl vtable* (Lane 2) bugs are very likely the **same
  imported-impl path** — assign them to the same worker, or sequence them. The *named func-value
  literal* bug also has one IR-gen site.
- **Lane 2 ⇄ Lane 3** share two cross-mode roots: **bit_cast sub-word narrowing** (native-aa64 facet
  in Lane 2, VM facet in Lane 3) and the (now-resolved) named-slice family's hardening tail. Same
  symptom, different fixes — keep them in their lanes but cross-reference findings.
- **Decision-gated (⚠) items are poor autonomous-bash candidates** — they need a language-semantics
  or scope ruling first. A worker hitting one should surface the decision rather than pick a behavior.
- **Large/deferred (🔶) items** (relro infra; VM bytecode `__Package`) are projects, not bashes —
  flag to the user before starting rather than absorbing them into a lane's sprint.
